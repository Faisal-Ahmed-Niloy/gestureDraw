How to Use

Gesture Drawing System — Quick Guide

01 · SETUP
1
Flash the ESP32

Upload main.cpp using PlatformIO. Check the Serial Monitor — it will print the ESP32's IP address once WiFi connects.

2
Set the IP in Python

Open gesture_drawing_client.py and update ESP32_IP to match the IP shown in Serial Monitor.

3
Start the bridge

Run python web_bridge.py in your terminal. This starts both the gesture engine and the local WebSocket bridge on port 8765.

4
Open this UI

Serve the folder with python -m http.server 5500 and visit http://localhost:5500 in your browser.

02 · GESTURES
☝️
DRAW
Index finger only — move your hand to paint strokes on the PC canvas.
DRAW
✌️
SEND
Peace / V-sign — pushes your drawing to the ESP32 display over WiFi.
SEND
🖐️
CLEAR
Open palm (4–5 fingers) — wipes the canvas on both PC and ESP32. Must draw first before clearing again.
CLEAR
✊
IDLE
Fist or any other hand shape — pen is lifted. No drawing or sending occurs.
IDLE
03 · KEYBOARD SHORTCUTS
QQuit the application
CForce-send drawing to ESP32 (same as V-sign)
XForce-clear ESP32 display and PC canvas
UUndo the last stroke
SToggle per-message debug logging in terminal
04 · TIPS
💡

Draw everything first, then send with a V-sign. This avoids unnecessary WiFi traffic during drawing.

💡

After clearing, you must draw at least one stroke before the palm-clear works again — this prevents accidental wipes.

💡

Use the Switch View button on the right panel to toggle between the ESP32 display mirror and your raw PC canvas.

💡

The green pill labelled WS: CONNECTED confirms the browser is talking to the Python bridge. If it turns red, restart web_bridge.py.

ABOUT THIS PROJECT

This project is a real-time gesture-based drawing system that bridges computer vision, embedded hardware and web technologies.

Hand gestures are detected via MediaPipe on a PC webcam, strokes are transmitted over WebSocket to an ESP32 microcontroller, and rendered live on an ILI9341 TFT display.

The web monitor (this UI) provides a live camera feed, an exact mirror of the ESP32 display, and real-time connection status — all driven by a lightweight Python bridge server.
